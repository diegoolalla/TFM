# Plantilla de Informe TFM

## Estructura del Informe (Máximo 20 páginas)

### 1. Portada
- Título del TFM
- Nombre del autor
- Nombre del máster
- Fecha
- Universidad/Institución

### 2. Resumen/Abstract (1 página)
- Breve descripción del problema
- Metodología empleada
- Principales resultados
- Conclusiones clave
- Palabras clave

### 3. Índice (1 página)

### 4. Introducción (2 páginas)
- Contexto y motivación
- Objetivos generales y específicos
- Alcance del proyecto
- Estructura del documento

### 5. Estado del Arte / Marco Teórico (2-3 páginas)
- Revisión de trabajos relacionados
- Conceptos teóricos relevantes
- Técnicas de Machine Learning utilizadas
- Justificación de las técnicas seleccionadas

### 6. Descripción del Dataset (1-2 páginas)
- Origen y características del dataset
- Número de instancias y características
- Descripción de variables
- Variable objetivo
- Licencia y consideraciones éticas

### 7. Metodología (3-4 páginas)

#### 7.1 Análisis Exploratorio de Datos
- Estadísticas descriptivas
- Distribuciones de variables
- Análisis de correlaciones
- Identificación de patrones

#### 7.2 Preprocesamiento de Datos
- Tratamiento de valores nulos
- Detección y manejo de outliers
- Codificación de variables categóricas
- Normalización/Estandarización
- División train/test

#### 7.3 Modelización
- Modelos seleccionados
- Configuración de hiperparámetros
- Estrategia de validación cruzada
- Métricas de evaluación

### 8. Resultados (4-5 páginas)

#### 8.1 Análisis Descriptivo
- Principales hallazgos del análisis exploratorio
- Visualizaciones clave
- Insights sobre los datos

#### 8.2 Rendimiento de Modelos
- Comparación de modelos
- Métricas de evaluación
- Matrices de confusión
- Curvas ROC/AUC
- Importancia de características

#### 8.3 Análisis de Errores
- Casos de fallo del modelo
- Análisis de falsos positivos/negativos

### 9. Discusión (2-3 páginas)
- Interpretación de resultados
- Comparación con trabajos previos
- Limitaciones del estudio
- Implicaciones prácticas
- Lecciones aprendidas

### 10. Conclusiones y Trabajo Futuro (1-2 páginas)

#### 10.1 Conclusiones
- Resumen de logros principales
- Cumplimiento de objetivos
- Contribuciones del trabajo

#### 10.2 Trabajo Futuro
- Mejoras propuestas
- Líneas de investigación futuras
- Aplicaciones potenciales

### 11. Referencias
- Bibliografía en formato estándar (APA, IEEE, etc.)
- Referencias a datasets, librerías y herramientas

---

## Anexos (Opcional, sin límite de páginas)

### Anexo A: Código Fuente
- Fragmentos de código relevante
- Enlace al repositorio GitHub

### Anexo B: Resultados Adicionales
- Tablas de resultados completas
- Visualizaciones adicionales
- Análisis complementarios

### Anexo C: Especificaciones Técnicas
- Entorno de desarrollo
- Hardware utilizado
- Versiones de software

## Consejos para la Redacción

### Formato
- Fuente: Times New Roman o Arial, 12pt
- Interlineado: 1.5 o 2.0
- Márgenes: 2.5 cm
- Numeración de páginas
- Títulos y subtítulos bien diferenciados

### Visualizaciones
- Todas las figuras deben estar numeradas
- Incluir títulos descriptivos
- Referenciar en el texto
- Alta calidad (mínimo 300 DPI)
- Etiquetas legibles

### Tablas
- Numeradas y con título
- Formato consistente
- Valores redondeados apropiadamente
- Referencias en el texto

### Estilo
- Redacción clara y concisa
- Voz activa cuando sea posible
- Tiempo verbal consistente
- Evitar jerga innecesaria
- Definir términos técnicos

### Revisión
- Verificar ortografía y gramática
- Revisar coherencia entre secciones
- Comprobar todas las referencias
- Validar números y cálculos
- Pedir feedback antes de entregar
